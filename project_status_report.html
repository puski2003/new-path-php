15. Known Issues / Bugs to Verify

These items may have bugs or are not end-to-end tested:
  • PayHere return URL — payment status only verified on browser return. If user closes browser, session may not be created. IPN callback needed.
  • Google Meet generation — requires valid OAuth2 refresh token in .env. Token expires; no auto-refresh error handling shown to user.
  • Booking hold release — check-on-read pattern means expired holds only clear when someone else tries to book. Consider a cron-style cleanup.
  • Community post image upload — verify max file size and MIME type validation to prevent malicious uploads.
  • Onboarding step redirect — verify users who completed onboarding aren't re-routed to onboarding on every login.
  • Password hashing — BCrypt $2a$ vs $2y$ prefix compatibility. Confirm PHP's password_verify() handles both for Java-migrated accounts.
Note on HTTPS: The proposal's legal feasibility section (§5.4) states all data must be encrypted in transit using TLS/SSL. XAMPP by default serves over HTTP. Before any demo or submission, enable SSL in XAMPP (httpd-ssl.conf) or deploy behind a reverse proxy. Assessors may check this.
